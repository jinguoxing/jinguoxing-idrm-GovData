1. **整体页面结构（信息架构）**

# 一、整体 UI 信息架构（Bottom-up 识别模块）

```
Bottom-up 识别任务
│
├─ ① 识别任务概览
│
├─ ② 识别结果对比（表 / 字段）
│    ├─ 表级对象建议
│    ├─ 字段语义分类
│    ├─ 状态 / 行为升级建议
│
├─ ③ 批量确认与生效
│
└─ ④ 冲突解释与校准
```

---

# 二、页面一：识别结果对比页（最重要）

> **这是用户停留时间最长的页面**

## 2.1 页面定位

**页面名称**

> 「识别结果确认」

**页面目标**

> 让用户快速判断：
> **“AI + 规则识别的结果是否符合业务认知”**

---

## 2.2 页面整体布局（左右对照式）

```
┌──────────────────────────────────────────────┐
│ 顶部：识别任务信息 / 过滤器 / 批量操作入口 │
├──────────────────────────────────────────────┤
│                                              │
│ 左侧：数据来源结构        右侧：语义建议    │
│                                              │
│ 表 / 字段列表            对象 / 属性 / 升级 │
│                                              │
└──────────────────────────────────────────────┘
```

---

## 2.3 顶部区域（全局控制）

### 元素

* 任务名称 / 数据源
* 筛选器：

  * 仅显示需确认
  * 仅显示冲突
  * 按置信度排序
* 入口按钮：

  * 【批量确认】
  * 【查看冲突】

---

## 2.4 左侧：数据来源结构区（事实视角）

### 展示内容

**表树结构**

```
订单表 order_info
 ├─ order_id
 ├─ user_id
 ├─ order_status
 ├─ pay_time
 └─ create_time
```

### 行为

* 点击表 → 切换右侧对象建议
* 点击字段 → 高亮对应语义建议

---

## 2.5 右侧：语义建议区（认知视角）

### 表级对象建议卡片

**卡片结构**

* 对象名称（AI 建议）
* 来源表
* 置信度（进度条）
* 风险提示（tag）
* 操作按钮：

  * ✔ 接受
  * ✎ 编辑
  * ✖ 拒绝

---

### 字段级语义分类列表

| 字段           | 语义角色 | AI 解释 | 操作 |
| ------------ | ---- | ----- | -- |
| order_id     | 标识   | 主键    | ✔  |
| order_status | 状态   | 枚举字段  | ✔  |
| pay_time     | 行为线索 | 支付发生  | ⚠  |

---

### 状态 / 行为升级建议（独立区块）

> **视觉上与普通字段区分（灰底 / 虚线框）**

```
⚠ 升级建议
- 「订单状态」→ 状态对象
- 「支付时间」→ 支付行为
```

---

# 三、页面二：批量确认页（效率核心）

> **解决“上百张表 / 上千字段”问题**

---

## 3.1 页面定位

**页面名称**

> 「批量确认」

---

## 3.2 页面结构（表格驱动）

```
┌──────────────────────────────────────────┐
│ 筛选区（类型 / 置信度 / 风险）           │
├──────────────────────────────────────────┤
│ 勾选 | 类型 | 建议内容 | 置信度 | 操作 │
│                                            │
│ [✓] | 表对象 | 订单 | 0.92 | 接受 |
│ [ ] | 字段 | order_status → 状态 | 0.75 | 查看 |
│                                            │
├──────────────────────────────────────────┤
│ 批量操作区                               │
│ [批量接受] [批量拒绝] [导出]             │
└──────────────────────────────────────────┘
```

---

## 3.3 批量规则（防误操作）

* 仅 **高置信度 + 无冲突** 项默认勾选
* 状态 / 行为升级 **不默认勾选**
* 批量操作前弹出确认说明

---

# 四、页面三：冲突解释与校准页（信任核心）

> **这是语义平台区别于普通自动建模工具的关键页面**

---

## 4.1 页面定位

**页面名称**

> 「冲突与解释」

---

## 4.2 页面布局（三栏式）

```
┌──────────────────────────────────────────────┐
│ 左：规则判断      中：AI 判断      右：决策 │
└──────────────────────────────────────────────┘
```

---

## 4.3 左栏：规则判断

* 命中规则列表
* 每条规则：

  * 规则编号
  * 命中原因
  * 权重影响

---

## 4.4 中栏：AI 判断

* AI 给出的分类结果
* 置信度
* 原文引用（字段名 / 备注）
* 推理说明（简述）

---

## 4.5 右栏：用户决策

* 推荐结果（高亮）
* 用户可选：

  * 采用规则
  * 采用 AI
  * 手动指定
* 决策说明（可选填写）

---

# 五、交互细节（必须强调）

## 5.1 不暴露内部术语

* UI 中不出现：

  * Entity / Event / Snapshot
  * Rule Score / AI Score

显示为：

* “对象”
* “字段含义”
* “业务结果 / 业务行为”

---

## 5.2 所有确认都有“可回溯”

* 每次确认：

  * 记录人
  * 时间
  * 依据（规则 / AI）

---

## 5.3 所有 AI 输出都有「为什么」

* 点击「为什么这么判断」
* 展开解释区

---

# 六、这套 UI 结构解决了什么问题

| 问题       | 如何解决   |
| -------- | ------ |
| AI 误判不敢用 | 冲突解释页  |
| 表多字段多    | 批量确认页  |
| 用户不懂建模   | 业务语言展示 |
| 模型污染     | 明确生效点  |
